# Cloud CDN

Una **CDN** es una red de servidores distribuidos geográficamente que **almacenan en caché** contenido (como imágenes, vídeos, archivos estáticos, etc.) y lo entregan desde el servidor más cercano al usuario final. Esto **reduce la latencia**, mejora el **rendimiento** y alivia la carga en tu servidor de origen.

Los contenidos estáticos (como imágenes, archivos CSS/JS, vídeos, HTML, etc.) se **almacenan temporalmente** en los **edge locations** (nodos perimetrales de Google alrededor del mundo). Esto se llama **caching**.

Cloud CDN es ideal cuando:

- Tienes una web pública con contenido estático
- Sirves archivos multimedia a usuarios globales
- Quieres escalar la entrega de contenido sin sobrecargar tu backend
- Quieres optimizar la experiencia de usuario globalmente

### 📦 **¿Solo contenido estático?**

**Principalmente sí**, pero *no exclusivamente*.

Cloud CDN **está optimizado para contenido estático**, como:

- Imágenes, CSS, JS, vídeos, PDFs, etc.
- Archivos servidos por aplicaciones backend si tienen headers apropiados (`Cache-Control`)

🔸 **Pero también puede cachear contenido dinámico**, **siempre que se configure correctamente**. Por ejemplo:

- Respuestas de una API que cambian poco o se actualizan cada cierto tiempo.
- HTML generado dinámicamente pero con contenido igual para todos los usuarios.

Eso sí, **no ejecuta lógica del lado del servidor**, solo distribuye contenido que se haya generado previamente.

Sí, **Cloud CDN (y las CDNs en general)** funcionan exclusivamente sobre el protocolo **HTTP/HTTPS** (es decir, el protocolo web).
